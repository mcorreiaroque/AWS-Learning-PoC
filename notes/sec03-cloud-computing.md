# Section 3: What is Cloud Computing?

## 1. Traditional IT vs. Cloud Computing
* **Traditional IT:** Like buying a house. You have high upfront costs (CapEx), you are responsible for maintenance, and scaling is slow.
* **Cloud Computing:** Like renting an apartment. You pay for what you use (OpEx), scaling is instant (Agility), and the provider handles the "heavy lifting" (maintenance).

## 2. Six Advantages of Cloud Computing
1. **Trade Capital Expense for Variable Expense:** Pay-as-you-go.
2. **Benefit from massive economies of scale.**
3. **Stop guessing capacity:** Avoid under-provisioning or over-provisioning.
4. **Increase speed and agility.**
5. **Stop spending money running and maintaining data centers.**
6. **Go global in minutes.**

## 3. Types of Cloud Computing
### Deployment Models:
* **Public Cloud:** Resources are owned and operated by a third-party (AWS).
* **Private Cloud:** Cloud resources used exclusively by a single organization (On-premises).
* **Hybrid Cloud:** Keeping some resources on-premises and some in the public cloud.

### Service Models:
* **IaaS (Infrastructure as a Service):** You manage the OS, Data, and Apps (e.g., EC2).
* **PaaS (Platform as a Service):** You only manage Apps and Data (e.g., Elastic Beanstalk).
* **SaaS (Software as a Service):** Everything is managed by the provider (e.g., Gmail, Dropbox).

## 4. Shared Responsibility Model
* **AWS:** Responsible for security **OF** the cloud (Hardware, Global Infrastructure, Regions, Edge Locations).
* **Customer:** Responsible for security **IN** the cloud (Data, IAM, Encryption, Firewall configuration).

## 5. Pricing Fundamentals
AWS has 3 main pricing drivers:
1. **Compute** (pay for compute time).
2. **Storage** (pay for data stored).
3. **Data transfer OUT** (Data transfer IN is free).
